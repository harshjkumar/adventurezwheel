# Adventures Wheel - Comprehensive Website Prompt

## 1. DESIGN THEME & VISUAL IDENTITY

### Color Palette
- **Primary Background**: Pure white (#FFFFFF) and soft off-white (#F8F9FA)
- **Accent Color**: Fresh green (#2ECC71, #27AE60, #1E8449) - used strategically throughout
- **Text Colors**: 
  - Dark charcoal (#2C3E50) for headings
  - Slate gray (#5D6D7E) for body text
  - White text on green backgrounds
- **Supporting Colors**:
  - Light green tints for backgrounds (#E8F8F0)
  - Subtle gray borders (#E5E7EB)
  - Success states in green
  - Error states in muted red

### Design Reference
Based on https://explorajourneys.com/int/en design language:
- Replace yellowish/gold tones with green equivalents
- Maintain the same luxurious, spacious layout
- Use similar full-width hero sections
- Implement identical card hover effects
- Replicate the smooth scrolling experience
- Mirror the navigation transparency effects
- Adopt the same typography hierarchy and spacing

### Typography
- **Primary Font**: Modern sans-serif (Inter, Montserrat, or Poppins)
- **Headings**: Bold weights (600-700), large sizes with generous letter-spacing
- **Body**: Regular weight (400), comfortable line-height (1.6-1.8)
- **Accent Text**: Medium weight (500) for CTAs and highlights

### Layout Principles
- Generous white space throughout
- Full-width sections with contained content (max-width: 1280px)
- Asymmetric layouts for visual interest
- High-quality imagery with subtle green overlays
- Glass-morphism effects on cards (white background with slight transparency)
- Soft shadows for depth (subtle, not harsh)

## 2. ANIMATIONS & INTERACTIONS

### Page Load Animations
- **Hero Section**: 
  - Fade in from bottom with 0.8s duration
  - Stagger animation for heading (0.2s delay), subtext (0.4s delay), CTA (0.6s delay)
  - Background image subtle zoom-in effect (scale 1.05 to 1.0 over 1.5s)

- **Content Sections**:
  - Scroll-triggered fade-in-up animations
  - Elements animate when 20% visible in viewport
  - Stagger child elements by 0.1s each
  - Use easing: cubic-bezier(0.4, 0.0, 0.2, 1)

### Navigation Animations
- **Navbar**:
  - Slide down from top on page load (0.5s)
  - Background blur and slight opacity change on scroll (white backdrop with 80% opacity + blur(10px))
  - Smooth color transition when scrolling past hero (0.3s transition)
  - Logo subtle pulse animation on hover (scale 1.05)
  - Mobile menu slides in from right with backdrop fade-in
  - Dropdown menus: fade-in with slide-down (0.2s)

### Card & Component Animations
- **Trip Cards**:
  - Hover effect: lift up (translateY(-8px)) with shadow increase (0.3s ease-out)
  - Image subtle zoom on hover (scale 1.1 over 0.5s)
  - Green overlay fade-in on hover (opacity 0 to 0.15)
  - Button slides in from bottom on card hover
  - Info text fades in with slight delay

- **Feature Cards**:
  - Rotate slightly on hover (rotate 1deg)
  - Icon bounce animation on hover
  - Background color shift (white to light green tint)

- **Gallery Images**:
  - Masonry grid layout with stagger load animation
  - Lightbox zoom-in animation (scale 0.9 to 1.0 with fade)
  - Image expand from clicked position with backdrop blur

### Button Animations
- **Primary CTAs**:
  - Background color slide animation (green gradient shift)
  - Ripple effect on click
  - Slight scale on hover (1.02)
  - Arrow icon slide-right on hover
  - Shadow expansion on hover

- **Secondary Buttons**:
  - Border color pulse animation
  - Text color transition
  - Background fill on hover (from transparent to light green)

### Scroll Animations
- **Parallax Effects**:
  - Hero background image moves at 0.5x scroll speed
  - Section backgrounds at 0.3x speed
  - Decorative elements at varying speeds for depth

- **Progress Indicators**:
  - Reading progress bar at top (green fill from left to right)
  - Scroll-to-top button: fade-in when scrolled 300px, rotate on hover
  - Smooth scroll behavior with easing

### Form Animations
- **Input Fields**:
  - Label float-up animation when focused
  - Border color change (gray to green) on focus
  - Validation checkmark slide-in from right
  - Error shake animation (3 quick horizontal movements)

### Loading States
- **Page Transitions**:
  - Skeleton loaders with shimmer effect (light gray to white gradient moving)
  - Spinner with green circular animation
  - Content fade-in after data load

- **Lazy Loading**:
  - Images fade-in when loaded (IntersectionObserver)
  - Blur-to-sharp transition for lazy images

### Micro-interactions
- **Icon Animations**:
  - Social icons: rotate and color change on hover
  - Search icon: expand to search bar
  - Heart icon: scale pulse when favorited
  - Star ratings: fill animation from left to right

- **Counter Animations**:
  - Stats numbers count-up animation when in viewport
  - Duration: 2s with easing
  - Format with comma separators

## 7. API ENDPOINTS

### Public APIs

**POST /api/auth/register**
- Create new user account
- Body: { email, password, name, phone }
- Returns: { success, user }

**POST /api/payment/create-order**
- Create Razorpay order
- Body: { amount, tripTitle, tripSlug, customerName, customerEmail, customerPhone, totalPayable, riders }
- Returns: { orderId, amount, currency, keyId }

**POST /api/payment/verify**
- Verify payment signature and save booking
- Body: { razorpay_order_id, razorpay_payment_id, razorpay_signature, bookingDetails }
- Returns: { success, verified, dbSaved, bookingId, paymentId, orderId }

**POST /api/contact**
- Submit contact form / enquiry
- Body: { name, email, phone, tripInterest, message }
- Returns: { success, enquiryId }

### Admin APIs

**GET /api/admin/trips**
- List all trips with relations
- Query: ?category=...&status=...
- Returns: { trips[] }

**POST /api/admin/trips**
- Create new trip
- Body: { trip data, itinerary[], pricing[], departures[] }
- Returns: { success, trip }

**GET /api/admin/trips/:id**
- Get single trip details
- Returns: { trip }

**PUT /api/admin/trips/:id**
- Update trip
- Body: { trip data, itinerary[], pricing[], departures[] }
- Returns: { success, trip }

**DELETE /api/admin/trips/:id**
- Soft delete trip (set is_active = false)
- Returns: { success }

**GET /api/admin/bookings**
- List all bookings
- Query: ?status=...&from=...&to=...
- Returns: { bookings[], totalRevenue, pendingAmount }

**PUT /api/admin/bookings/:id**
- Update booking status
- Body: { booking_status, payment_status }
- Returns: { success }

**GET /api/admin/enquiries**
- List enquiries
- Query: ?status=...
- Returns: { enquiries[] }

**PUT /api/admin/enquiries/:id**
- Update enquiry status
- Body: { status }
- Returns: { success }

**GET /api/admin/blogs**
- List all blog posts
- Returns: { blogs[] }

**POST /api/admin/blogs**
- Create new blog post
- Body: { title, slug, excerpt, content, featured_image, author_name, category }
- Returns: { success, blog }

**PUT /api/admin/blogs/:id**
- Update blog post
- Body: { blog data, is_published }
- Returns: { success }

**DELETE /api/admin/blogs/:id**
- Delete blog post
- Returns: { success }

**GET /api/admin/gallery**
- List gallery images
- Returns: { images[] }

**POST /api/admin/gallery**
- Upload new image
- Body: FormData with image file
- Returns: { success, image }

**DELETE /api/admin/gallery/:id**
- Delete gallery image
- Returns: { success }

**GET /api/admin/homepage**
- Get homepage sections
- Returns: { heroSlides[], stats[], howItWorks[], testimonials[] }

**PUT /api/admin/homepage**
- Update homepage sections
- Body: { section_name, items[] }
- Returns: { success }

## 8. RESPONSIVE DESIGN BREAKPOINTS

### Mobile First Approach
- **Mobile**: < 640px
  - Single column layouts
  - Hamburger menu
  - Stacked cards
  - Full-width images
  - Bottom-fixed booking widget

- **Tablet**: 640px - 1024px
  - 2-column grids
  - Side drawer menu
  - Card grid with 2 columns

- **Desktop**: > 1024px
  - 3-4 column grids
  - Horizontal mega menu
  - Sidebar filters
  - Sticky elements

- **Large Desktop**: > 1280px
  - Max content width: 1280px
  - Centered layouts
  - Enhanced spacing
